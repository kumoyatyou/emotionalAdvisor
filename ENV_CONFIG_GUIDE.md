# ⚙️ 环境变量 (.env) 高阶与性价比配置指南

在项目根目录中，`.env` 文件是整个 Agent 的大脑中枢。
这篇文档的核心目的是：**教你如何通过为不同的 Skill 分配最合适的模型，来实现“既聪明又省钱”的极致性价比；同时，以国内优质聚合平台“硅基流动”为例，提供一份手把手的保姆级配置教程。**

---

## 💡 核心思路：为什么不同 Skill 要配不同模型？

我们的 Agent 拥有多项核心技能（Skill），每个技能对模型的能力要求完全不同。如果全部用最贵的最强模型，遇到几万字的微信聊天记录时，Token 消耗会让你非常心疼；但如果全用免费/廉价模型，复杂的业务逻辑又容易出错。

因此，根据每个 Skill 的特点“因材施教”才是最划算的。以下是针对各模块最适合的**模型能力标签**与推荐：

*   **全局主模型 (`MODEL_NAME`) & 情感军师 (`SIMP_MODEL`)**
    *   **🏷️ 模型标签要求**：`[长文本处理]` `[高性价比]` `[情感细腻]`
    *   **工作特点**：需要处理海量的聊天记录（长文本上下文），对情感的理解要细腻，但不需要极其复杂的数理逻辑。
    *   **高性价比选择**：推荐 Kimi (如 `Pro/moonshotai/Kimi-K2.5`) 或 Qwen 系列 (`Qwen/Qwen2.5-72B-Instruct`)。它们上下文窗口大，且处理日常聊天非常便宜。
*   **角色模拟 (`NUWA_MODEL`)**
    *   **🏷️ 模型标签要求**：`[角色扮演]` `[指令遵循]` `[表现力强]`
    *   **工作特点**：需要极强的“扮演感”和语气模仿能力（比如模仿马斯克或张雪峰），必须能严格遵循系统 Prompt，不能轻易出戏。
    *   **高性价比选择**：推荐 `Pro/deepseek-ai/DeepSeek-V3`，它在角色扮演和指令遵循上表现极其出色且价格合理。
*   **八字算命 (`BAZI_MODEL`)**
    *   **🏷️ 模型标签要求**：`[知识提取]` `[精准表达]` `[无过度发散]`
    *   **工作特点**：**注意，这个 Skill 内部已经自带了严密的八字排盘算法和命理书籍知识库！** 因此，它**不需要**模型自己去进行复杂的逻辑推理（使用推理模型反而容易脱离提供的知识库去“过度发散”或胡言乱语）。它只需要一个能精准理解输入知识、并按指令流畅输出的模型。
    *   **高性价比选择**：推荐使用满血的通用大模型，例如 `Pro/deepseek-ai/DeepSeek-V3` 即可，完全不需要上具备强推理能力（如 R1 级别）的慢速模型。
*   **知识库向量 (`EMBEDDING_MODEL`)**
    *   **🏷️ 模型标签要求**：`[文本向量化]` `[低成本]` `[稳定]`
    *   **工作特点**：只用来将文字转化为多维向量存入 Chroma 数据库，不参与对话生成。
    *   **高性价比选择**：越便宜越好。推荐 `Qwen/Qwen3-Embedding-8B`（在硅基流动等平台上通常极其便宜甚至免费）。

## 🚀 手把手实战：以“硅基流动”为例的配置教程

“硅基流动”(SiliconFlow) 是国内优秀的 API 聚合平台，集合了 DeepSeek、Qwen、Kimi 等大量顶级国产模型，不仅网络稳定不需要科学上网，且很多小模型完全免费，非常适合作为本项目的首选后端。

以下是详细的配置步骤：

### 第一步：获取 API Key
1. 访问硅基流动官网（[siliconflow.cn](https://siliconflow.cn/)）并注册账号。
2. 登录后，进入左侧导航栏的 **“API 密钥”** 菜单。
3. 点击“新建 API 密钥”，复制生成的以 `sk-` 开头的那一长串字符，妥善保存。

### 第二步：查找准确的模型名称
1. 在左侧导航栏进入 **“模型广场”**。
2. 挑选你想要的模型，点击进去后，可以在标题下方看到类似 `Pro/deepseek-ai/DeepSeek-V3` 或 `Qwen/Qwen2.5-72B-Instruct` 的字符串，旁边通常有个“复制”按钮，这就是我们需要填入 `.env` 的模型全名。

### 第三步：填写 `.env` 文件
在你的项目根目录找到 `.env` 文件（如果没有，就把 `.env.example` 复制一份并重命名为 `.env`），然后按照你的实际需求进行配置。

### 第四步：测试与运行
保存 `.env` 文件后，重新启动你的 `launcher.py` 或 `gui_main.py`。

## 🌐 扩展：如何配置其他大模型平台？

由于本项目底层采用了兼容 OpenAI 的接口规范，所以理论上支持**任何提供 OpenAI 兼容格式 API** 的模型平台。
你只需要修改 `.env` 文件中的 `OPENAI_API_BASE` 和 `OPENAI_API_KEY` 即可无缝切换。

以下是几个常见主流平台的配置指南：

### 1. 官方 DeepSeek 平台
DeepSeek 官方的接口价格极其厚道，且能力顶级。
- **OPENAI_API_BASE**: `https://api.deepseek.com` （注意：有些客户端需要加 `/v1`，但官方标准推荐这个）
- **OPENAI_API_KEY**: 你的 DeepSeek 官方 API Key
- **常见模型名称**: `deepseek-chat` (对应 V3), `deepseek-reasoner` (对应 R1)

### 2. 官方 OpenAI
如果你有顺畅的网络环境和官方 API。
- **OPENAI_API_BASE**: `https://api.openai.com/v1`
- **OPENAI_API_KEY**: 你的官方密钥 (`sk-...`)
- **常见模型名称**: `gpt-4o`, `gpt-4o-mini`

### 3. 本地私有化部署 (如 Ollama)
如果你显卡性能不错，希望数据 100% 不出本机，可以使用 Ollama 在本地跑模型。
- **OPENAI_API_BASE**: `http://127.0.0.1:11434/v1` (Ollama 的默认兼容接口地址)
- **OPENAI_API_KEY**: `sk-123` (本地部署不需要鉴权，随便填一个即可，但不建议留空)
- **常见模型名称**: `qwen2.5:14b`, `llama3:8b` (必须填你通过 `ollama run` 下载到本地的准确模型名)

### 4. 其他第三方 API 中转站
如果你购买了第三方的 API 代理服务。
- **OPENAI_API_BASE**: 中转服务商提供的地址（通常以 `/v1` 结尾，例如 `https://api.xxxx.com/v1`）
- **OPENAI_API_KEY**: 中转商给你的 Token
- **常见模型名称**: 视中转商支持的模型列表而定，一般与官方名称一致。
